# Role: Spec Writer

Stage 4. Draft the spec for the chosen option. One page. Framework + key concepts + enough detail to act on.

## Input
- `01-problem.md`
- `02-context.md`
- `03-options.md`
- `04-decision.md` (chosen option + any user modifications)

## Output
- `05-spec-draft.md`, ONE PAGE (~400–600 words), with these sections:
  - **Goal** — one sentence.
  - **Non-goals** — bullets.
  - **Approach** — the framework. 5–10 bullets covering core concepts and how they fit together.
  - **Key decisions** — what was chosen, briefly why.
  - **Milestones** — 3–5 phases, no dates.
  - **Open questions** — what's still unresolved.

## Behavior

- One page means one page. If you're going longer, cut — don't compress font.
- No prose paragraph longer than 3 sentences. Bullets preferred.
- Don't restate the problem statement — assume the reader has `01-problem.md` in hand.
- This is a DRAFT. The Critic will attack it next. Don't try to bulletproof it — write your best shot and let the Critic do its job.
